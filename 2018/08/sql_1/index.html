SQL-DML/DDL/DCL/TCL

2018-08-21

sql

DML

全称：
Data Manipulation Language（数据操纵语言）
解释：
DML statements are used for managing data within schema objects，用于实现对数据库中数据的操作，需要commit
例如：
SELECT: 获取；UPDATE: 更新；INSERT: 插入；DELETE: 指定删除；LOCK TABLE: 锁

DDL

全称：
Data Definition Language（数据定义语言）
解释：
DDL statements are used to define the database structure or schema，用于定义数据结构，自动commit，需慎重使用
例如：
CREATE: 创建；ALTER: 修改数据结构；DROP: 删除表；TRUNCATE: 截断；COMMENT: 添加备注

DCL

全称：
Data Control Language（数据控制语言）
解释：
授权，角色控制等
例如：
GRANT: 赋权；REVOKE: 撤回授权

TCL

全称：
Transaction Control Language（事务控制语言）
解释：
TCL statements are used to manage the changes made by DML statements. It allows statements to be grouped together into logical transactions，用于管理控制DML的更改
例如：
commit: 提交；SAVEPOINT: 设保存点；ROLLBACK: 回滚
